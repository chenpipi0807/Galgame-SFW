# 如何创建一个好的 GG 卡

> 以「董卓模拟器」这类复杂可玩卡为例，讲清楚一张优秀 GG 卡的内核到底由什么构成，以及在创建向导里该怎么写提示词，才能让 AI 一次性生成「角色设计 + 任务线索 + 世界规则 + 双模式可玩」都立得住的故事。

---

## 一、先理解 GG 卡的「四层内核」

一张卡好不好玩，不取决于文笔，而取决于**结构信息是否齐全且互相咬合**。GalGame 把一张卡拆成四层数据（对应数据库与系统提示词的真实字段），你写概念时心里要装着这四层：

| 层 | 真实字段 | 作用 | 谁来驱动 |
|---|---|---|---|
| **① 世界层** | `concept` / `vibe` / `opening` / `backstory` | 故事基调、开场镜头、世界观 | 玩家可读，AI 叙事的底色 |
| **② 规则层** | `mechanics` / `phases` / `key_items` | 核心机制、阶段推进条件、胜负判定、关键道具 | 阶段管理器 + 叙述者 |
| **③ 角色层** | `faction` / `secret_objective` / `starting_items` / `starting_location` / `aliases` | 阵营对立、秘密任务、初始物品与站位 | NPC 行为的「内心 OS」 |
| **④ 动态层** | 9 个记忆槽位（NPC 状态 / 阵营势力 / 地点 / 关系网 / 悬念线索 / 关键事件 / 重要物品…） | 游戏过程中实时演化的状态 | 记忆管理器每 10 条自动更新 |

**关键认知：玩家在阅读页只直接看到 ① 层，但真正让卡「耐玩、有张力、能反转」的是 ②③ 层这些隐藏字段。**「董卓模拟器」之所以复杂好玩，就是因为它把规则层和角色层填得很满——每个 NPC 都有阵营和私密目标，世界有明确的阶段和胜负条件。

> 一句话原则：**好卡 = 一个有矛盾的世界（①②）× 一群有私心的人（③）× 会随你行动变化的状态（④）。**

---

## 二、创建流程怎么和这四层对应

6 步向导里，你的输入主要落在第 ① 步「概念」和第 ⑤ 步「角色」，但你写的概念会被 AI 用来**反推出全部四层**。所以概念写法决定上限。

```
① 概念  ─┐
② Vibe  ─┼─→ plot_creator → ①世界层 + ②规则层(mechanics/phases/key_items)
③ 开场  ─┘
④ 背景
⑤ 角色  ───→ character_gen → ③角色层(每个NPC的阵营/秘密任务/物品/站位)
⑥ 确认  ───→ 生头像、落库，可玩
```

- 第 ② 步 **Vibe**（甜蜜 / 悬疑 / 权谋 / 奇幻…）决定 `plot_creator` 的情感光谱切换。董卓这类选「权力较量 / 悬疑博弈」。
- 第 ③④ 步的「选项一」AI 会**严格遵从你的输入**，不发散。所以你写得越具体，世界越是你想要的样子。
- 第 ⑤ 步会**全量提取**故事中每个角色，不限数量，并为每人补齐阵营和秘密任务。

---

## 三、写「初始概念」的核心技巧

这是最重要的一步。下面每条技巧都给出「弱写法 → 强写法」的对比。

### 技巧 1：给一个有内在矛盾的「局」，而不是一个场景

矛盾 = 可玩性的发动机。最好让玩家一进场就身处多方角力中。

- 弱：`我穿越成了董卓。`
- 强：`我穿越成东汉权臣董卓，刚入主洛阳。手握西凉铁骑却根基未稳：朝中士族貌恭心异、并州吕布勇而易叛、关东诸侯正暗结联盟。我要在巩固权力与众叛亲离之间走钢丝。`

强写法直接喂给 AI 三股对立势力 + 一个核心张力（巩固 vs 失控），`plot_creator` 自然会生成阵营、机制和胜负判定。

### 技巧 2：明示「身份 + 资源 + 威胁 + 目标」四件套

这是让 AI 生成规则层的最稳模板：

```
我是【身份/视角角色】，
拥有【初始资源/筹码：兵力、官职、金钱、信息、特殊能力】，
但面临【威胁/限制：敌对势力、时间压力、隐藏规则】，
我的目标是【可判定的胜利条件】。
```

「目标」一定要**可判定**（能称帝 / 活过三个回合 / 集齐五枚虎符），这样 `phase_manager` 才能写出终局条件。模糊目标（「体验人生」）会让卡缺少推进力。

### 技巧 3：主动埋「阶段感」

如果你希望故事有节奏推进，在概念里暗示时间线或里程碑，AI 会据此生成 `phases`（如董卓的 `["入主洛阳", "废立天子", "迁都长安", "群雄并起"]`）。

- 触发阶段的写法：`故事从…开始，随着…，最终走向…。`

### 技巧 4：点名几个关键角色 + 他们的「立场」

不用写全，但点 2~4 个核心 NPC，并暗示其阵营倾向，AI 会自动补秘密任务：

- `身边有：义子吕布（勇猛但重利易叛）、谋士李儒（忠诚阴狠）、司徒王允（表面归顺暗藏杀机）、貂蝉（身份成谜）。`

`character_gen` 看到「暗藏杀机 / 身份成谜」就会给王允、貂蝉填上对立 `faction` 和 `secret_objective`，这是「反转」的来源。

### 技巧 5：声明世界规则 / 特殊设定

如果你的世界有非常规规则（异能、数值、轮回、死亡惩罚），在概念里直接写成「规则条款」，AI 会放进 `mechanics`，也方便你之后补成**世界书条目**：

- `规则：每个回合结束结算"威望值"，低于 0 立即被刺杀；持有传国玉玺可强制压制一名诸侯。`

## 四、把「任务线索」做厚：世界书 + 隐藏字段

复杂卡的「耐玩」来自玩家不断挖出新信息。两个工具：

### 1. 角色隐藏字段（开局即埋）

第 ⑤ 步生成后，去**世界书弹窗 → 角色**标签检查/补全每个 NPC：

- `faction`：确保**至少存在两个对立阵营 + 一个摇摆方**，单一阵营 = 没有博弈。
- `secret_objective`：每个重要 NPC 都该有一个**和玩家不完全一致**的私密目标（王允：诛董；吕布：谋取最大利益；貂蝉：连环计）。这是 NPC 在「经典模式」里敢拒绝、敢背叛你的依据。
- `starting_items` / `starting_location`：给关键道具和站位，`phase_manager` 的「搜证 / 对峙」判定要用到。

### 2. 世界书（Lorebook / 关键词触发）

把「查得到的背景知识」做成**关键词 → 规则内容**的条目。玩家或 NPC 提到关键词时自动注入给 AI，等于一张张「线索卡」。

- 例：关键词 `["传国玉玺","玉玺"]` → 内容「藏于何处、有何效果、谁在觊觎」。
- 优先级原则（系统已内置）：实时状态 > 阶段机制 > 世界观 > 背景 lore；记忆槽位优先于 lorebook。
- 建议给复杂卡建 10~30 条，覆盖：重要人物秘史、地点机关、道具规则、历史伏笔。
- 可用 `Ctrl+F` 在游戏中模糊搜索全部条目，方便自查是否有遗漏或冲突。

> 线索设计口诀：**每条线索要么指向一个目标，要么解锁一个能力，要么暴露一个秘密。** 纯背景设定不算线索。

## 五、把「世界规则」做硬：mechanics 与 phases

规则层是「剧本杀感」的来源，让故事不只是聊天而是**有输赢的局**。

写好规则层，确保 `mechanics` 里能回答三个问题：

1. **怎么推进？**——阶段转换条件（如「至少一名诸侯起兵 → 进入群雄并起」）。
2. **怎么赢/输？**——胜负判定（威望归零被刺 / 称帝 / 阵营目标达成）。
3. **关键道具怎么用？**——`key_items` 的效果与获取方式。

这些可以在概念里埋（技巧 2/3/5），也可以创建后在**世界书**里补成条目。`phase_manager` 会读记忆槽位自动判断是否进入下一阶段并产出转场叙事（钟声、战报、广播）。

## 六、怎么把卡做得耐玩

GalGame 采用**经典**系统提示词（剧本杀 RPG，NPC 有独立意志会对抗）。一张耐玩的卡，靠的是**信息分层**：让规则层和角色层都厚实。

| 层 | 吃哪层数据 | 设计要点 |
|---|---|---|
| **冲突骨架** | 重度依赖 ②规则层 + ③阵营/秘密任务 | 矛盾越尖锐、阵营越对立、秘密越多，越好玩 |
| **角色血肉** | 重度依赖 ①基调 + ③角色性格/关系 | 角色魅力、关系张力、前史要足 |

**做法：**

1. **角色既要有「立场」也要有「魅力」。** `personality` 不要只写功能性（「忠诚的谋士」），加入情感维度（「忠诚阴狠，却对你有复杂的执念」），让角色立体可信。
2. **关系网要同时承载冲突与张力。** 貂蝉对董卓「既是连环计的棋子，又心生真实摇摆」，矛盾与情感并存才有戏。
3. **规则层永远值得填满。** 阵营/秘密任务是剧情的骨架，越完整越能撑起长线叙事。
4. **Vibe 多选。** 同时勾「权谋 + 悬疑」，让 `plot_creator` 多种基调都注入底色。

> 反过来说：只写「人」不写「局」的卡会很空；只写「局」不写「人」的卡会很干。**两层都厚 = 卡才立得住。**

## 七、董卓模拟器式复杂卡 · 创建清单

照着这张清单填，基本能产出一张高复杂度、双模式可玩的卡：

- [ ] **概念**用「身份 + 资源 + 威胁 + 目标」四件套写，含一个核心张力。
- [ ] 目标**可判定**，并暗示**3~4 个阶段**的时间线。
- [ ] 点名 **4~6 个核心 NPC**，每人暗示阵营倾向或秘密。
- [ ] **Vibe 多选**，覆盖博弈层 + 情感层。
- [ ] 生成后检查角色层：**≥2 个对立阵营 + 摇摆方**，每个重要 NPC 有**秘密任务**。
- [ ] `personality` 兼具**立场 + 情感魅力**两个维度。
- [ ] 补 **10~30 条世界书**线索（每条指向目标 / 解锁能力 / 暴露秘密）。
- [ ] 在 mechanics/世界书里写清**推进条件、胜负判定、关键道具规则**。
- [ ] 关键道具配好 `key_items` 与 NPC `starting_items`/`starting_location`。
- [ ] 两种模式各试玩几轮，回看是否「局」与「人」都立得住，缺哪层补哪层。

## 八、一个可直接套用的「概念」模板

```
我穿越/成为【身份】，身处【时代/世界 + 一句话局势】。
我拥有【资源A、资源B、特殊能力C】，
但【威胁X 在外、隐患Y 在内、限制Z 缠身】。
身边有：【角色1（性格+立场暗示）】、【角色2（性格+秘密暗示）】、
        【角色3（魅力+摇摆暗示）】……
世界规则：【非常规规则1；非常规规则2】。
故事从【起点事件】开始，随着【中段冲突】，最终走向【可判定的终局】。
我的目标是【胜利条件】，但每个人都有自己的盘算。
```

把方括号填满，向导生成出来的卡就会自带阵营、秘密、阶段和机制——这正是「董卓模拟器」复杂度的配方。
