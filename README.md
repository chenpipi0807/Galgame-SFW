# GalGame

> **GalGame（旮旯给木）—— 高自由度、深度沉浸的剧本杀级 AI 叙事引擎。**
> 角色拥有独立意志，世界规则随剧情生长，每一局都是不可复制的沉浸式剧场。

GalGame 是一个完全本地运行的 AI 沉浸式叙事 / 角色扮演 Web 应用。通过 6 步 AI 创作向导生成完整故事世界（开场、背景、角色），以流式 NDJSON 协议驱动多角色叙事，配备结构化记忆槽位、关键词触发的世界书注入和异步实况生图。所有故事（剧情 + 角色 + 对话 + 记忆 + 世界书）可导出为 AES-256 加密的 PNG「GG 卡」—— 一张图就是一个可跨设备分享的存档。

技术栈：Python FastAPI + SQLite（后端）/ Vue 3 + Vite（前端）。

> **"你的私人叙事宇宙"** —— 创建、扮演、沉浸、分享。

## 快速开始

### 环境要求

- Python 3.11+
- Node.js 18+

### 1. 安装依赖

```bash
# 后端
cd backend && pip install -r requirements.txt

# 前端
cd frontend && npm install
```

### 2. 配置 API（零配置开箱即用）

**默认内置了一个免费的 Agnes 连接（LLM + 生图都能用），不填任何 Key 就能直接玩。**

> ⚠️ 这个 Agnes Key 是项目内置的**公共 Key**，大家共用。如果遇到限流/失效，或想要更好的文笔与画质，请在「管理后台 → API 配置」**添加你自己的连接**（DeepSeek / gpt-image-2 等高级模型体验更好）。

API 配置改为**「连接列表」模型** —— 每个「连接」= 一个 API 来源（名字 + 地址 + Key + 模型）。启动后进入「管理后台 → API 配置」：

- 🎭 **叙事模型（LLM）** / 🎨 **生图模型** 各是一个连接列表，顶部「＋ 添加连接」。
- 每条连接可 **启用 / 测试 / 编辑 / 删除**；每类同时只有一个「使用中」。
- **LLM** 一律走 OpenAI 兼容 `/chat/completions`（DeepSeek、各类代理都属于此）。
- **生图** 选类型：`OpenAI 兼容生图`（gpt-image / dall-e / 多数代理）或 `Gemini`。
- 你的连接（含 Key）只存在**本机数据库**，绝不上传，也不会进 git。

> **🔑 想换更好的模型？常见 Key 获取：**
> - **DeepSeek**（推荐 LLM，上下文 128K、中文顶级）：[platform.deepseek.com/api_keys](https://platform.deepseek.com/api_keys)
> - **AIHubMix**（推荐生图，gpt-image-2 模型）：[aihubmix.com](https://aihubmix.com)
> - **Agnes**（免费无限 LLM + 生图，可注册自己的）：[platform.agnes-ai.com/settings/apiKeys](https://platform.agnes-ai.com/settings/apiKeys)
> - **Gemini**（生图可选）：[aistudio.google.com/apikey](https://aistudio.google.com/apikey)

### 3. 一键启动

```bash
python start.py
```

浏览器打开 [http://localhost:15927](http://localhost:15927)

- 后端端口：17832
- 前端端口：15927
- `Ctrl+C` 同时关闭前后端

## 完整功能指南

### 🏠 故事库

首页展示你所有的故事，每个故事一张卡片。

| 操作 | 怎么用 |
|------|--------|
| **新建故事** | 点击「＋ 新建故事」进入 6 步创建向导 |
| **修改标题** | 直接点击故事卡片上的标题文字 → 修改 → 点 ✓ 确认 |
| **编辑故事** | 悬停卡片，点左上角 ✎ 图标 |
| **删除故事** | 悬停卡片，点左上角 ✕ 图标 → 确认删除（不可恢复） |
| **开始阅读** | 已发布/待发布的故事，点「马上穿越」进入 |
| **继续创作** | 草稿状态的故事，点「捏个世界」回到向导 |
| **导入 GG 卡** | 点「📥 导入 GG 卡」选择 PNG → 自动恢复完整故事 |

**故事状态说明：**
- 🔴 **草稿** — 未完成创建向导
- 🟡 **待发布** — 已完成设定，可阅读但未正式发布
- 🟢 **已发布** — 正式发布，可导出 GG 卡分享

### ✨ 创建向导

6 步 AI 辅助创作，从概念到完整故事世界。

> 📝 **想做出「董卓模拟器」那种角色、任务线索、世界规则都立得住的复杂卡？** 先读 [《如何创建一个好的 GG 卡》](如何创建一个好的GG卡.md) —— 手把手讲清概念提示词写法与四层内核设计技巧。

| 步骤 | 内容 | 你可以做什么 |
|------|------|-------------|
| **① 概念** | 输入一句话描述故事 | 也支持「📎 上传角色卡」从酒馆 PNG 一键导入 |
| **② Vibe** | 勾选情感标签（甜蜜/悬疑/奇幻…） | 多选，决定故事基调 |
| **③ 开场** | AI 生成 3 个开场白（首个严格遵从用户概念） | 选一个喜欢的，或点「✏️ 自己写」 |
| **④ 背景** | AI 生成 3 个世界观设定（首个严格遵从用户已选开场） | 同上 |
| **⑤ 角色** | AI 生成角色卡片（含阵营、秘密任务、初始物品） | 可修改外貌/性格/阵营，上传参考图，指定玩家角色；所有出场角色全量提取不限制数量 |
| **⑥ 确认** | 等待头像生成 | 全部完成后自动跳转阅读页 |

> 💡 **酒馆卡导入**：在第 ① 步上传 SillyTavern PNG 角色卡，AI 自动提取角色信息、开场白、世界观。或在第 ⑤ 步从背景设定页导入。

#### 世界书弹窗

点「🌐 世界书」打开，4 个标签页：

| 标签 | 内容 |
|------|------|
| **故事设定** | 修改标题/Vibe/开场白/背景设定 |
| **角色** | 批量编辑所有角色（名字/外貌/性格/别名/生图描述） |
| **世界书** | 管理 Lorebook 条目（见下方） |
| **系统提示词** | 查看当前模式使用的提示词（可在管理后台在线编辑） |

#### 世界书（Lorebook / RAG）

> 💡 **核心概念**：当你输入的消息中包含世界书条目的**关键词**时，对应的**规则内容**会自动注入给 AI。
> 支持优先级排序（实时状态 > 阶段机制 > 世界观 > 背景 lore）、冲突解决（记忆槽位优先于 lorebook）、按阶段动态注入。

| 操作 | 怎么做 |
|------|--------|
| **导入世界书** | 「📤 上传 JSON」→ 支持 SillyTavern 世界书、角色卡预设、任意 JSON 格式 |
| **新建/编辑条目** | 手动添加关键词 + 规则内容，逐条可折叠 |
| **搜索** | 搜索框即时过滤大量条目 |
| **快速查阅** | 游戏中按 `Ctrl+F` 弹出搜索面板，模糊搜索当前故事所有世界书的**关键词和内容**，结果高亮匹配片段 |
| **挂载/卸载** | 编辑器内可将世界书挂载到故事，聊天内也可临时附加 |

#### 记忆系统

每 10 条消息自动更新一次记忆槽位。点「🧠 记忆」随时查看，点「刷新」手动更新。

| 槽位 | 追踪内容 |
|------|---------|
| 👤 玩家角色 | 当前状态、位置、经历 |
| 🌆 当前场景 | 地点、时间、阶段（入席/搜证/对峙/终局）、氛围 |
| 💫 NPC 状态 | 各 NPC 当前状态、位置、态度、阵营、暴露程度、任务进度 |
| 🏛 阵营势力 | 各阵营成员、胜利条件、当前势力状态 |
| 📍 地点状态 | 各地点在场角色、物品、状态（开放/上锁/有密道） |
| 💕 关系网 | 角色间关系类型与张力 |
| 🔍 悬念线索 | 进行中的悬疑线索 |
| 📅 关键事件 | 已发生的重要事件及其影响 |
| 🎁 重要物品 | 关键道具、持有者、用途 |

#### 🌱 角色演变 & 世界规则生长

记忆系统不仅记录**状态**，还会让角色与世界**随剧情真正生长**——这是区别于"设定写死"的核心玩法：

- **角色性格/背景演变**：当某角色发生**持久性的本质转变**（黑化、信念崩塌、身世揭穿、立场反转）时，系统自动把这层变化**追加**到该角色的演变层，并在后续叙事中**优先于原始设定**生效。原始底设永久保留、不被覆盖。
- **世界规则生长**：剧情中**新出现**的世界设定/规则会被自动收进「📜 剧情演变」世界书，按关键词在后续叙事里触发。
- **省 token**：演变搭车记忆更新调用，无额外开销；只有发生实质变化的角色才写入，绝大多数轮次零增量。
- **可感知**：演变角色的头像会标 🌱 徽标，点开角色详情卡可见「🌱 性格/背景演变」区块。
- **触发时机**：跟随记忆更新（每 10 轮自动，或点「🧠 记忆 → 存档」手动触发）。
- **重玩即重置**：点「🗑 清空对话」会一并清空演变层、演变日志与自动生成的世界规则，回到角色初始设定。

### 🛠 故事编辑器

从故事库卡片悬停 ✎ 进入（或 `/edit/:id`），3 个标签页：

#### 故事设定
- 编辑标题、概念、开场白、背景设定、叙事视角（第一/二/三人称）
- 「发布故事」按钮：将草稿/待发布状态改为已发布

#### 角色
- 每个角色独立卡片，编辑名字/外貌/性格/生图描述/风格
- 「重新生成」头像
- 「导入酒馆角色卡」从 PNG 追加角色
- 「＋ 添加角色」手动新增

#### 世界书
- 管理 Lorebook，与阅读页世界书弹窗的编辑功能一致
- 「挂载世界书」将已有世界书关联到当前故事

### ⚙️ 管理后台

访问 `/admin` 或阅读页 → 世界书 → 系统提示词 → 「在后台编辑」。

| 标签 | 功能 |
|------|------|
| **API 配置** | 「连接列表」管理：叙事(LLM) / 生图各一组连接，添加 / 启用 / 测试 / 编辑 / 删除；内置免费 Agnes，开箱即用 |
| **提示词编辑** | 在线编辑所有系统提示词（叙述者/角色生成/记忆管理/实况生成/猜你喜欢） |
| **任务监控** | 查看后台异步任务状态（生图/记忆更新），失败可重试 |

> 💡 提示词编辑后自动生效（10 秒 TTL 缓存），无需重启服务。

## 技术栈

| 层 | 技术 |
|---|---|
| 前端 | Vue 3 + Vite + Tailwind CSS + Pinia + Vue Router |
| 后端 | Python 3.11+ + FastAPI + aiosqlite |
| LLM | Agnes / DeepSeek（流式输出） |

| 存储 | SQLite（单文件本地数据库） |

## 数据说明

运行时数据保存在 `data/`（已加入 `.gitignore`）：

- `galgame.db`：SQLite 数据库（故事、角色、会话、消息、设定）
- `uploads/`：用户上传与 AI 生成的图片
- `logs/`：运行日志（`galgame.log`）

API「连接」配置（含自填的第三方 Key）存于本机数据库 `data/galgame.db` 的 `providers_config`（由 `backend/core/providers.py` 管理），内置免费 Agnes Key 仅硬编码在代码中。所有配置均不上传任何服务器、也不进 git。
